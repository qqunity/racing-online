# 🏎️ Racing Online

Браузерная 2D-гонка на машинках с мультиплеером. Создавай комнату или заходи по
коду, шашкуй между потоком трафика стрелками **←/→**, собирай **нитро** ⚡,
объезжай **лужи масла** ☣ и приезжай к финишу первым.

## Как устроено

- **Клиент** — [Phaser 3](https://phaser.io/) (Vite). Сцены: меню → лобби →
  гонка → результаты.
- **Сервер** — Node.js + Express + [Socket.IO](https://socket.io/). Держит
  комнаты в памяти, авторитетно решает старт гонки, проверяет время финиша и
  строит итоговую таблицу.
- **Модель мультиплеера** — параллельные дорожки. Сервер выдаёт всем игрокам
  один **seed**, и клиент с сервером генерируют по нему **идентичную** раскладку
  трафика и бонусов (`shared/`). Все встречают одинаковые препятствия — разница
  только в скилле.
- **Один origin** в продакшене: сервер раздаёт собранный клиент и обслуживает
  WebSocket на том же порту.

```
shared/   общий детерминированный код (rng, генерация трассы, константы)
client/   Phaser-клиент (Vite)
server/   Express + Socket.IO
tests/    Playwright E2E (два browser context = два игрока)
```

## Локальный запуск (dev)

```bash
npm install
npm run dev
```

- сервер: http://localhost:3000
- клиент (Vite, с проксированием WebSocket на сервер): http://localhost:5173

Открой http://localhost:5173 в двух вкладках: в одной создай комнату, во второй
зайди по коду, затем у хоста нажми «Старт».

## Прод-сборка локально

```bash
npm run build   # собирает client/dist
npm start       # сервер раздаёт client/dist + WebSocket на :3000
# открой http://localhost:3000
```

## Тесты (Playwright E2E)

```bash
npm run test:install   # один раз: скачать Chromium
npm test               # соберёт прод-версию, поднимет сервер и прогонит сценарии
```

Покрытие: синхронизация лобби между двумя игроками, полный цикл гонки до
таблицы результатов, и проверка честности (одинаковый seed → идентичная трасса).

## Docker

```bash
docker compose up --build
# открой http://localhost:3000
```

Образ многоступенчатый: сначала собирается клиент, затем в рантайм-образ
кладётся только сервер + статика. Подходит для любого хостинга с поддержкой
WebSocket (Render, Railway, Fly.io, VPS). Порт настраивается через `PORT`.

## Управление

- **← / →** (или **A / D**) — смена полосы.
- Столкновение с трафиком — авария и потеря скорости.
- ⚡ **нитро** — ускорение. ☣ **лужа масла** — занос и потеря управления.
