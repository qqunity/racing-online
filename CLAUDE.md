# CLAUDE.md

Guidance for working in this repository.

> **Поддерживай этот файл в актуальном состоянии.** При любом изменении
> архитектуры (новый слой, смена стека, новый сетевой протокол/события,
> переезд сборки или деплоя) — обнови соответствующие разделы CLAUDE.md в том
> же изменении. Расхождение между кодом и этим файлом хуже его отсутствия.

## Что это

Браузерная 2D-мультиплеерная гонка на машинках. Игрок создаёт комнату или
заходит по коду, шашкует стрелками **←/→** между потоком трафика, собирает
нитро ⚡ и объезжает лужи масла ☣; побеждает тот, кто первым доедет до финиша.

## Стек и архитектура

- **Клиент** — Phaser 3 + Vite (`client/`). Сцены: Boot → Menu → Lobby → Race
  → Result. Меню/лобби/результаты — HTML-оверлеи (`client/src/ui/dom.js`) с
  атрибутами `data-testid` (нужны для Playwright). Ядро геймплея —
  `client/src/scenes/RaceScene.js`.
- **Сервер** — Node.js + Express + Socket.IO (`server/`). `rooms.js` —
  `RoomManager` (комнаты в памяти, коды, лобби, переназначение хоста).
  `raceManager.js` — жизненный цикл гонки (seed, отсчёт, прогресс, ранжирование,
  анти-чит на время финиша). `index.js` в проде раздаёт `client/dist` и
  обслуживает WebSocket на **том же порту** (один origin, без CORS).
- **`shared/`** — код, общий для клиента и сервера. `rng.js` (`mulberry32`),
  `track.js` (генерация трассы по seed), `constants.js`. **Фундамент честности:**
  сервер выдаёт один seed на комнату, и обе стороны детерминированно строят
  идентичную раскладку трафика/бонусов. Менять алгоритм генерации можно только
  синхронно для клиента и сервера.
- **`tests/`** — Playwright E2E. Два browser context = два игрока в одной
  комнате. `tests/fixtures.js` — хелперы (создание/вход/старт/финиш).

### Ключевые инварианты

- `shared/` импортируется и в браузере (Vite alias `@shared`), и в Node (ESM).
  У `shared/package.json` обязан быть `"type": "module"` — иначе в Docker-образе
  (без корневого package.json) файлы загрузятся как CommonJS и упадут.
- В `RaceScene` есть тест-хук `window.__GAME__` (`phase`, `distance`, `seed`,
  `fingerprint`, `autoFinish()`). На него опираются E2E-тесты — не ломай контракт.
- Socket.IO протокол: client→server `createRoom/joinRoom/startRace/progress/
  finished/leaveRoom`; server→client `roomCreated/roomJoined/roomUpdate/
  joinError/raceStarting/opponentProgress/raceResults/playerLeft`. При изменении
  событий правь обе стороны и этот список.

## Команды

```bash
npm install            # установка зависимостей всех workspaces
npm run dev            # dev: сервер :3000 + клиент Vite :5173 (с ws-прокси)
npm run build          # сборка клиента в client/dist
npm start              # прод: сервер раздаёт client/dist на :3000
npm run test:install   # один раз: скачать Chromium для Playwright
npm test               # E2E: соберёт прод, поднимет сервер (:3100), прогонит тесты
docker compose up --build   # прод в контейнере на :3000
```

## Очистка ресурсов (обязательно!)

Не оставляй после себя запущенные процессы и контейнеры — они занимают порты
(3000/3100/5173) и ломают следующий запуск.

- **Dev/прод-сервер, запущенный вручную:** останови его. Если стартовал в фоне —
  `pkill -f "node server/src/index.js"` (или по конкретному `PORT=...`).
- **Docker:** после проверки всегда `docker compose down` (останавливает и
  удаляет контейнер и сеть). Образ можно удалить `docker rmi racing-online`.
- **Playwright** сам поднимает и гасит сервер через `webServer`; в режиме
  `reuseExistingServer` (локально) может переиспользовать уже запущенный — следи,
  чтобы на нужном порту не висел чужой процесс.
- **Браузерные страницы chrome-devtools**, открытые для визуальной проверки,
  закрывай по завершении.
- Перед тем как считать задачу выполненной, убедись, что порты свободны:
  `lsof -i :3000 -i :3100 -i :5173` должен быть пустым.
